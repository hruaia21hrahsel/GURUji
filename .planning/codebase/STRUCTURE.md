## Directory Purposes

**app/ (Expo Router):**
- Purpose: All routable screens and API endpoints (file-based routing)
- Contains: `.tsx` screen files, API route files (`.+api.ts`)
- Key files: `_layout.tsx` (root), `(tabs)/_layout.tsx` (tab config), `api/chat+api.ts` (server proxy)

**app/(tabs)/ (Tab Navigator):**
- Purpose: Three main app screens after onboarding
- Contains: Home, Chat, Settings screens
- File pattern: One screen per `.tsx` file matching route name

**app/onboarding/ (Onboarding Stack):**
- Purpose: 3-step user setup (class → board → language)
- Contains: Three sequential screens
- File pattern: Route-based files (`index.tsx`, `board.tsx`, `language.tsx`)

**app/api/ (Server Routes):**
- Purpose: Backend API endpoints
- Contains: Only `chat+api.ts` initially (proxies to Claude)
- Pattern: Expo Router API routes use `+api.ts` suffix, execute server-side only

**components/:**
- Purpose: Reusable UI components (not full screens)
- Contains: Smaller components (buttons, cards, inputs, chat elements)
- Organization: Group by feature (e.g., `chat/`, `common/`)

**lib/:**
- Purpose: Shared utilities, types, configuration
- Contains: System prompts, Supabase client, type definitions, constants
- Key files:
  - `system-prompt.ts`: Full GURUji persona + `buildSystemPrompt(profile)` function
  - `supabase.ts`: Lazy Supabase client instance
  - `types.ts`: Shared TypeScript interfaces (Profile, Message, etc.)
  - `constants.ts`: Fixed lists (class options, board options, languages)

**store/:**
- Purpose: Zustand state management with AsyncStorage persistence
- Contains: Store definitions with `persist` middleware
- Key files:
  - `useAppStore.ts`: User profile + onboarding state
  - `useChatStore.ts`: Current chat session messages

**assets/:**
- Purpose: Static images and icons
- Contains: App icon, splash screens, adaptive icons for Android
- Generated by: Expo (some files auto-generated from icon.png)

## Key File Locations

**Entry Points:**
- `app/_layout.tsx`: Root layout (Expo Router entry point, NativeWind setup, onboarding guard)
- `app/api/chat+api.ts`: Server-side API endpoint for chat streaming

**Configuration:**
- `app.json`: Expo app metadata (name, bundle ID, icon, splash)
- `tsconfig.json`: TypeScript compiler settings (strict mode, path aliases)
- `tailwind.config.js`: Tailwind colors, theme extensions
- `babel.config.js`: Babel plugins (NativeWind JSX transform)
- `metro.config.js`: Metro bundler config (NativeWind integration)

**Core Logic:**
- `lib/system-prompt.ts`: GURUji persona and grade-level instruction logic
- `store/useAppStore.ts`: Profile management and onboarding state
- `store/useChatStore.ts`: Chat message history and session cache

**Screens:**
- `app/(tabs)/index.tsx`: Home/dashboard screen
- `app/(tabs)/chat.tsx`: Chat interaction screen (primary feature)
- `app/(tabs)/settings.tsx`: User settings and profile adjustment
- `app/onboarding/index.tsx`: Class selection
- `app/onboarding/board.tsx`: Board selection
- `app/onboarding/language.tsx`: Language selection

## Naming Conventions

**Files:**
- Screen files: lowercase with `.tsx` extension matching route (e.g., `chat.tsx`)
- Component files: PascalCase with `.tsx` extension (e.g., `ChatMessage.tsx`)
- Store files: `use[StoreName].ts` pattern (e.g., `useAppStore.ts`)
- API routes: `[name]+api.ts` pattern (e.g., `chat+api.ts`)
- Utilities: lowercase with `.ts` extension (e.g., `system-prompt.ts`)

**Directories:**
- Route groups: parentheses `(groupName)` for Expo Router organization
- Feature grouping: lowercase plural (e.g., `components/chat/`)
- Core directories: lowercase plural standard names (`app`, `components`, `lib`, `store`, `assets`)

**Exports:**
- Named exports: Function/component names in PascalCase (e.g., `export function ChatMessage()`)
- Store exports: Default export of hook (e.g., `export default useAppStore`)
- Constants: UPPERCASE_SNAKE_CASE (e.g., `export const DEFAULT_CLASS = "6"`)

## Where to Add New Code

**New Feature (e.g., Subject Quick-Start):**
- Screen: `app/(tabs)/[feature-name].tsx` or add to tab navigator in `(tabs)/_layout.tsx`
- Components: `components/[feature-name]/Component1.tsx`, `components/[feature-name]/Component2.tsx`
- State: Add reducer/action to relevant store (or create `store/use[Feature]Store.ts` if isolated state)
- Types: Add interfaces to `lib/types.ts`

**New Component/Module:**
- Reusable UI: `components/[category]/ComponentName.tsx`
- Screen: `app/[route]/page.tsx` (Expo Router convention)
- Utility/Helper: `lib/[utility-name].ts`

**New API Route:**
- Server endpoint: `app/api/[endpoint-name]+api.ts` (e.g., `app/api/feedback+api.ts`)
- Validation: `lib/validation.ts` (server-side only)
- Types: Server request/response types in `lib/types.ts`

**Utilities (Shared Helpers):**
- String/number helpers: `lib/utils.ts`
- Date formatting: `lib/date.ts`
- Subject-specific logic: `lib/subjects/[subject].ts` (e.g., `lib/subjects/math.ts`)
- Formatting: `lib/formatting.ts`

**Styling:**
- Global styles: `global.css` (Tailwind directives)
- Component-scoped: Use `className` prop with Tailwind classes in components
- Custom theme: Extend `tailwind.config.js` for new colors/spacing

## Special Directories

**app/api/:**
- Purpose: Server-only API endpoints (Expo Router convention)
- Generated: No
- Committed: Yes
- Note: Files with `+api.ts` suffix execute server-side only; ANTHROPIC_API_KEY accessible here

**.planning/codebase/:**
- Purpose: Analysis and planning documents
- Generated: Yes (by `/gsd:map-codebase` command)
- Committed: Yes
- Contains: ARCHITECTURE.md, STRUCTURE.md, STACK.md, CONVENTIONS.md, TESTING.md, CONCERNS.md

**assets/:**
- Purpose: Static resources (images, icons)
- Generated: Partially (Expo generates some adaptive icons)
- Committed: Yes (all source assets)

**node_modules/:**
- Purpose: Dependency packages
- Generated: Yes (by npm install)
- Committed: No (in .gitignore)

**expo/:**
- Purpose: Expo dev cache and compiled bundles
- Generated: Yes (by expo start)
- Committed: No (in .gitignore)
