**lib/auth.ts:** Extend the file created in Plan 01. Add all sign-in functions:

1. `signInWithPhone(phoneNumber: string)` — Validates E.164 format (must start with +), calls `supabase.auth.signInWithOtp({ phone: phoneNumber })`, returns `{ data, error }`.

2. `verifyPhoneOtp(phone: string, token: string)` — Calls `supabase.auth.verifyOtp({ phone, token, type: 'sms' })`, returns `{ data, error }`.

3. `signInWithGoogle()` — Configures `GoogleSignin` with `webClientId` from `EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID` env var. Calls `GoogleSignin.hasPlayServices()`, then `GoogleSignin.signIn()`. Takes the `idToken` from the response and calls `supabase.auth.signInWithIdToken({ provider: 'google', token: idToken })`. Returns `{ data, error }`. Wraps in try/catch to handle Expo Go graceful failure (Google Sign-In crashes in Expo Go — catch and return friendly error).

4. `signUpWithEmail(email: string, password: string)` — Validates email format (basic regex), validates password length >= 6 chars. Calls `supabase.auth.signUp({ email, password })`. Returns `{ data, error }`.

5. `signInWithEmail(email: string, password: string)` — Calls `supabase.auth.signInWithPassword({ email, password })`. Returns `{ data, error }`.

6. `requestPasswordReset(email: string)` — Calls `supabase.auth.signInWithOtp({ email })`. Returns `{ data, error }`.

7. `verifyEmailOtp(email: string, token: string)` — Calls `supabase.auth.verifyOtp({ email, token, type: 'email' })`. Returns `{ data, error }`.

All functions are async and return `{ data, error }` matching Supabase's pattern. On success, the `onAuthStateChange` listener in root layout (set up in Plan 01) automatically updates Zustand.
